# How Embeddings and Vector Database Were Used

The core idea behind this recommendation system is converting each song into a numeric vector that captures its meaning, not just its exact words. For every song in the dataset, I combined the genre, mood, artist, and description into one text string. This text was then passed through a pre-trained sentence embedding model called all-mpnet-base-v2 (from HuggingFace's sentence-transformers), which turns the text into a high-dimensional vector — basically a long list of numbers that represents what the song is "about."

The reason this works better than simple keyword matching is that the embedding model understands context and meaning. Two songs described differently in words but similar in feeling — say, one calling itself "soft romantic ballad" and another "gentle love song" — end up with vectors that are close to each other in this vector space, even though they don't share exact words.

Once every song had its embedding, I stored all of them in a Chroma vector database. Chroma is built specifically to store and search through large numbers of vectors quickly, using similarity search instead of exact matching. When a user selects a song, its embedding is used as the search query, and Chroma calculates the cosine similarity between that vector and every other song vector in the database. The songs with the closest vectors are returned as the top recommendations, along with a similarity score showing how close the match is.

This same setup is what makes the natural language search feature possible. When someone types something like "suggest relaxing songs for studying," that sentence is embedded the same way as the songs were, and compared against the entire database. Since the model understands meaning rather than literal words, it can correctly match a query like "relaxing" to songs tagged "Chill" or described using calm, peaceful language, even if the word "relaxing" never appears anywhere in the dataset.

On top of this similarity search, I added simple filters for genre, mood, language, and release year. These don't involve the AI model at all — they just check the metadata stored alongside each song and remove any results that don't match what the user selected. This combination of semantic search (for finding songs that feel similar) and metadata filtering (for narrowing down by exact category) is what makes the final recommendations both smart and controllable.
